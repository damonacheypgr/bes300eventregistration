Synchronous Version

This demonstration is showing the user experience of adding someone as a participant to an event in a synchronous manner.

At the server side, when you POST to /events/id/participants route the server makes a call to the employees resource to verify they are still an active employee.

We block the UI while this is happening because our state is unknown in the client. Is the person registered? Are they not registered?
